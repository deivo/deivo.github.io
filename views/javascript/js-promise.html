Deivo 8/29/2019 JavaScriptInterview

# Promise

  • # Promise

    • Promise诞生主要目的： 解决 "ajax" 中串行和并行，以及在AJAX的串行和并行中产生的回调地域问题，它是一种设计模式，ES6中提供了一个JS内置类Promise，来实现这种设计模式。

    • 回掉地域 -> 上一个回调函数中继续做事情，而且继续回调（在真实项目的AJAX请求中经常出现回调地域）=> 异步请求、不方便代码的维护

      /student?class=1 获取班级信息
/score?stuID=[1,2,3] 返回学员信息
/jige?score=90 确定是否及格

$.ajax({
    type: "get",
    url: "/student",
    data: {
        class: 1
    },
    success: function (response) {
        $.ajax({
            type: "get",
            url: "/score",
            data: {
                stuID: [1,2,3]
            },
            success: function (response) {
                $.ajax({
                    type: "get",
                    url: "/jige",
                    data: {
                        score: 90
                    },
                    success: function (response) {
                    
                    }
                });                 
            }
        });
    }
});

基于Promise来优化上面的代码
function ajax1() {
    new Promise(resolve=>{
        type: "get",
        url: "/student",
        data: {
            class: 1
        },
        success: resolve
    }) 
}
function ajax2(arr) {
    new Promise(resolve=>{
        type: "get",
            url: "/score",
            data: {
                stuID: arr
            },
        success: resolve
    }) 
}
function ajax3() {
    new Promise(resolve=>{
        type: "get",
        url: "/jige",
        data: {
            score: 90
        },
        success: resolve
    }) 
}
ajax.then(result => {

    return ajax1();

async 和 await 的语法糖

    let result = ajax1();

    let result = ajax3();

    • 总结

        1. 回调地域，代码难以维护，第一个的输入是第二个的输入， 基于Promise的链式调用
        1. Promise可以支持多个并发的请求，获取并发请求中的数据
        1. 这个Promise可以解决异步的问题，本身不能说Promise是异步的

    • 特性： Promise是用来管理异步编程的，他本身不是异步的，new Promise的时候会立即把executor函数执行（只不过我们一般会在executor函数中处理一个异步操作），会给[executor]函数传递两个实参信息：resolve函数、reject函数

    • Promise三个状态：
      1. pending 初始状态； value: undefined
      2. fulfilled(现在叫resolved) 代表操作成功； value: 传递的第一个实参
      3. rejected 代表操作失败； value 传递的第一个实参
      • Promise本身还有一个Value值，用来记录成功的结果（或失败的结果）
      let p1 = new Promise(()=>{
    console.log(1) 
})
console.log(2)
以上先输出1，在输出2

let p1 = new Promise(()=>{
    setTimeout(_=>{
        console.log(1) 
    })
})
console.log(2)
以上先输出2，在输出1

let p1 = new Promise(()=>{
    setTimeout(_=>{
        console.log(1) 
    })
    console.log(2)
})
console.log(3)
以上先输出2，在输出3, 最后输出1

    • resolve函数和reject函数

      1. 一般会在异步操作结束后，执行resolve/reject函数，执行这两个函数中的一个，都可以修改Promise的status状态（【[PromiseStatus]】）和value值（【[PromiseValue]】）。 
        let p1 = new Promise((resolve, reject)=>{
    setTimeout(_=>{
        resolve('ok');
    },1000)
})

      1. 只要Promise的状态被改变，再执行resolve、reject就没有用了。 
        let p1 = new Promise((resolve, reject) => {
    setTimeout(_ => {
        console.log(10);
        resolve('ok');
        console.log('ok');
        reject('no');
        console.log('no');
    }, 1000)
})
p1.then(result=>{
    console.log("@成功", result);
},reason=>{
    console.log("@失败", reason);
})
// 10 ok no @成功10

      1. resolve函数或reject函数的执行，不论是否放到一个异步操作中，都需要等待then先执行完，把方法存储好，才会在更改状态后执行then中对应的方法 => 此处是一个异步操作（所以很多人说Promise是异步的），而且是微任务操作resolve(100)。 

        let p1 = new Promise((resolve, reject) => {
    setTimeout(_ => {
        if(Math.random < 0.5){
            reject('NO!');
            return;
        }
        resolve("ok")
    }, 1000)
})
p1.then(result=>{
    console.log(result);
}.reason => {
    console.log(reason);
})

      2. 基于 then 方法我们可以向当前实例对应的“事件池”中存放零到两个方法，如果Promise的状态变为成功，会把 then 中存放的第一个方法执行，如果状态是失败，则THEN中存放的第二个方法执行；不论执行那个方法都会把 VALUE值 传递给方法.

        let p1 = new Promise((resolve, reject)=>{
    console.log(1);
    resolve(100);
    console.log(2);
})
p1.then(result=>{
    console.log(`成功：`+ result);
},reason => {
    console.log(`失败` + reason)
})

      1. new Promise创建出来的实例，成功或者失败，取决于 executor 函数执行的时候，执行的是 resolve函数执行 还是 reject函数 ；或者executor函数执行发生异常错误，会把当前实例状态修改为 失败。 
        let p1 = new Promise((resolve, reject)=>{
    throw new Error('错误');
    resolve(100);
});
// p1实例状态变为失败

      1. p2/p3这种每一次执行返回的新实例的状态，有then中存储的方法执行的结果来决定最后的状态（上一个then中某个方法执行的结果，决定下一个then中哪一个方法会执行）

        1. 不论是成功的方法执行，还是失败的方法执行（then中两个方法），凡事执行抛出异常，则都会把实例的状态改为失败。
        2. 方法中如果返回一个新的Promise实例,返回这个实例。
        3. 剩下的情况基本上都是让实例变为成功的状态(方法返回的结果是当前实例的value值：上一个then中返回的结果传递给下一个then的方法中)。
        then方法结束都会返回一个新的Promise实例(then链)
【[PromiseStatus]】: 'pending'
【[PromiseValue]】: 'undefined'

let p1 = new Promise((resolve,reject)=>{
    resolve(100);    
})
let p2 = p1.then(result =>{
    console.log(`成功：`+ result);
    return 200;
},reason => {
    console.log(`失败` + reason)
    return -10;
})
let p3 = p2.then(result =>{
    console.log(`成功：`+ result);
},reason => {
    console.log(`失败` + reason)
});

      2. 创建一个成功或者失败的Promise实例的两个方法 Promise.rosolve(1); Promise.reject(1); 

        new Promise((resolve,reject)=>{
    resolve(100);
})
上面的写法也可以被这种代替：
Promise.resolve(100)代表创建一个成功的Promise实例
Promise.reject(100)代表创建一个失败的Promise实例

    • then中也可以只写一个或者不写函数

      • then方法结束都会返回一个新的Promise实例(then链)。
      • 遇到一个then，要执行成功或者失败的方法，如果此方法并没有在当前then中被定义，则顺延到下一个对应的函数 ---- 真实项目中，一般不这么搞，都是用catch
        Promise.reject(10).then(result => {
    console.log(`成功：${result}`)
}).then(null, reason=>{
    console.log(`失败：${result}`)
})
根据顺延规则，返回结果就是 "失败 10"
      • Promise.prototype.catch(fn) 等价于 .then(null, fn)
        Promise.resolve(10).then(result=>{
    console.log(a);
}).catch(reson=>{
    console.log(`失败：${reson}`)
})
      • then方法中只写一个或者不写函数

    • # Promise.all(arr):

      • 返回的结果是一个Promise实例(all实例) ，要求arr数组中的每一项都是新的Promise实例，Promise.all是等待所有数组中实例状态都会成功才会让『all实例』状态为成功，value是一个集合，存储这arr中每一个实例返回的结果；凡是arr中有一个实例状态失败，『all实例』的状态也是失败

        • 返回结果：
          1. 如果有一个失败，那么Promise.all的state状态就是失败，value值就是'undefined'
          2. 如果成功，返回值的顺序是按照arr数组中编写的实例顺序组合在一起的
        let p1 = Promise.resolve(1);
let p2 = new Promise(resolve=>{
    setTimeout(_=>{
        resolve(2);
    },1000)
});
let p3 = Promise.reject(3);
Promise.all([p1,p2,p3]).then(result=>{
    console.log(`成功：${result}`);
}).catch(reason=>{
    console.log(`失败：${reason}`);
})
返回结果：失败 3

Promise.all([p1,p2]).then(result=>{
    console.log(`成功：${result}`);
}).catch(reason=>{
    console.log(`失败：${reason}`);
})
返回结果：一秒后返回：'成功：1,2' 

Promise.all([p2,p1]).then(result=>{
    console.log(`成功：${result}`);
}).catch(reason=>{
    console.log(`失败：${reason}`);
})
返回结果：一秒后返回：'成功 2,1'

    • # Promise.race(arr):

      • Promis.all不同的地方，race是赛跑，也就是arr中不管哪一个先处理完，处理完的结果作为『race实例』的结果

  • # async / await

    • 概念：async 和 await 是Promise的语法糖，ES 7新增的。
    • 真实项目中很少用.then和.catch，原因：因为ES7当中提供了Promise操作的语法糖: async / await

    • # async

      • async 是对函数一个修饰：使一个函数返回Promise实例。
      • 说明： async修饰后的函数，使函数返回一个Promise实例，Promise实例的status状态为[[resolved]]，Promise实例的value值为[[函数本身return的值，如果没有return，value值就是undefined]]，函数执行返回的Promise实例可以基于then方法来获取Promise实例的value值。
      • async起始就是放一个普通函数变为异步，函数里面是异步与函数没有关系
        async function fn() {
    return 10;
}
console.log(fn()); // 返回Promise实例

async function fn() {
    setTimeout(_=>{
        return 10;
    },1000)
}
console.log(fn()); // 返回Promise实例

以上这种方法项目中很少用
      • 作用： 主要是配合await使用的，因为一旦函数中使用await，那么当前的函数必须用async来修饰。
        let p1 = new Promise((resolve, reject) => {
    resolve(1);
})
async function fn() {
    let result  = await p1;
    console.log(result); // 1
}
fn();
      • 注意： async修饰后的函数，函数中使用了await，await的实例状态必须是为成功，才会去执行其后面的代码（[[状态为失败，后面代码不执行]]）
        let p1 = new Promise((resolve, reject) => {
    reject('0');
})
async function fn() {
    let result  = await p1;
    console.log(result); // 此行代码不执行，因为await p1的实例是失败状态
}
fn();

    • # async配合await使用注意事项:

      1. await会等待当前Promise的返回结果，只有返回的状态是Promise情况，才会把返回结果赋值给result
      2. await不是同步编程，是异步编程（微任务）:当代码执行到此行（下面let），构建一个异步的微任务（等待Promise返回结果,并且await下面的代码也都被放到任务队列中）
      3. 如Promise是失败状态，则await不会接收其返回结果，await下面的代码也不会在执行([[await只能处理Promise为成功状态的时候]])
        let p1 = Promise.resolve(100);
let p2 = new Promise(resolve=>{
    setTimeout(_=>{
        resolve(200);
    },1000)
});
let p3 = Promise.reject(3);
async function fn() {
    console.log(1);
    let result = await p1;
    console.log(result);
}
console.log(2)
返回结果： 1，2，100

async function fn() {
    console.log(1);
    let result = await p2;
    console.log(result);
    let result = await p1;
    console.log(result);
}
console.log(2)
返回结果： 1，2，200，100 

async function fn() {
    let reason = await p3;
    console.log(reason);
}
返回结果：无

# JS 面向对象

# 引言

在自然界，万事万物都是我们研究的对象，所有要研究和学习操作都是研究对象，把抽象的事物，按照特点进行戏份，分成大小和小类等等，在根据共同点把公共的进行抽取，归并到对应的类别中；对象只是一个泛指。类是对对象的一种细分，而实例是类别中派生出来的具体事物。从而来说，JS 也存在这种现象，JS 中对象也是我们学习和研究的。

# JS 中的对象

在 JS 中通常创建对象方式有两中，一种是通过new Object的内置类来创建，二种是通过字面量的方式创建，第二种创建的方式在每天的开发中，时时刻刻都在使用，这就是 JS 中的对象。

  • new方式创建：
let person = new Object()
person.name = 'Deivo'
person.age = '18'
person.like = function () {
  return `${this.name} likes football`
}

person.like()
  • 字面量方式创建：
const person = {
  name: 'Deivo',
  age: '18',
  like: function () {
    return `${this.name} likes football`
  }
}
person.like()

# JS 内置类

  • JS 中每一种数据类型都有一个自己对应的类别，如Number类，每一个数字（NaN、Infinity、-Infinity）都是Number类的实例
  • 内置类String、Boolean、（null、undefined浏览器保护起来了）、Symbol、BigInt、Object(Array、Math...)、Function...
  • 在每一个 DOM 元素对象，也有自己的类和祖先类，如：
    • DIV 元素对象 -> HTMLDivElement -> HTMLElement -> Element -> Node -> EventTarget -> Object
    • A 元素对象 -> HTMLAnchorElement -> HTMLElement ...
    • document -> HTMLDocument -> Document -> Node ...
    • window -> Window -> WindowProperties -> EventTarget ...
    • 元素集合 -> HTMLCollection -> Object
    • 节点集合 -> NodeList -> ...

# 构造函数

# 描述

  • 在 JS 中，通过new关键字来调用函数，称之为构造函数。
  • 构造函数的首字母一般必须大写，此时的函数也会被称之为类。
  • 构造函数在没有return情况下，返回值：对象，称之为此构造函数的一个实例，函数体中的this指向当类的实例
  • 通过 new 方式派生出类和实例

# 类

  • 类型： 函数数据类型

# 实例

  • 实例： 对象数据类型（每一个对象（包含实例对象）都有很多属性和方法，在自己堆内存中存储的都是私有的属性方法，基于__proto__原型链查找类prototype原型上的都是共有的属性方法）；

注意

  • 如果手动 return 的是一个值，对返回的实例无影响，
  • 如果手动 return 的是引用类型的值，会把默认返回的实例给替换掉（因此在构造函数模式执行下，我们一般都不会手动写 return，防止把返回的实例给替换）

# instanceof

  • 用来检测某个实例是否属于这个类型
  • 语法： 实例 instanceof 类
  • 返回值： 属于返回 true，不属于返回 false
  • 局限性：
    • 要求检测的实例必须是对象数据类型
    • 基本数据类型的实例是无法基于它检测出来的
function User (name, password) {
  this.userName = name
  this.password = password
}

let user = new User('deivo', '111111')
console.log(user instanceof Array)

# 基本数据类创建方式

  • 字面量创建方式
let n = 10
console.log(typeof n) // number
console.log(n.toFixed(2)) // 调用Number类原型上toFixed方法
  • 构造函数创建
let m = new Number('10')
console.log(typeof m) // "object"
console.log(m.toFixed(2)) // 调用Number类原型上toFixed方法
  • 通过以上两种方式，可以看到两种创建的基本类，数据类型是不一样。

# 注意

构造函数执行，因为也具备普通函数执行的特点，和实例有关系的操作一定是：this.x = x，因为this是当前类创建出来的实例，构造函数执行形成的私有变量和实例没有必然的联系

function Fn (n) {
  let m = 10
  this.total = n + m
  this.say = function () {
    console.log(this.tatal)
  }
}
let f1 = new Fn(10)
let f2 = new Fn(20)
let f3 = new Fn()
// new的时候不论是否加小括号，都相当于把Fn执行了,也创建了对应的实例，
// 只不过不加小括号是不能传递实参数的(当前案例中的形参n=undefined)

console.log(f1.m) // => undefined  构造函数的两个特点
console.log(f2.n) // => undefined  构造函数的两个特点
console.log(f1.total) // => 20
console.log(f2.say()) // => this:f2(因为say执行前面有点)
console.log(f1 === f2) // =>false (因为是两个不同的实例对象，也就是不同的堆地址)

# 构造函数执行底层机制

  • new 执行函数，具有普通函数执行的特点
    • new 执行也会类当作普通函数执行（当然也是类执行的一面）
      1. 创建一个私有的上下文
      1. 形参赋值 & 变量提示
      1. 浏览器先创建一个对象出来（这个对象就是 当前类的一个新实例），并且让函数this指向这个实例对象
      1. 代码执行
      1. 在不设置 return 的情况下，浏览器会把创建的实例对象默认返回
function Fn () {
  this.x = 100
  this.y = 200
}
let f1 = new Fn()
let f2 = new Fn()
  • 上面代码中 f1和f2都是类Fn执行创建出来的实例

# 原型

  • 函数数据类型： 普通函数、类（内置类和自定义类）、箭头函数(ES6 新增)都是Function这个类的实例

  • 对象数据类型： 普通对象、数组对象、正则对象、日期对象、实例对象、函数对象（函数也是一个对象，也想普通对象一样，有自己的属性）、类的prototype都是Object这基类的实例

  • 1. 每一个函数（[[除 ES6 的箭头函数外]]）都有一个内置的属性 prototype(原型属性)，属性值是一个对象，在对象中会存储当前类的公共属性和方法。

  • 2. 在 prototype 这个对象堆内存中，如果是浏览器为其默认开辟的堆内存，会存在一个内置的属性：constructor 构造函数，属性值就是当前类本身

// 通过上面两句话
function Fn () {}
Fn.prototype.constructor === Fn // true
  • 3. 每一个对象都有一个内置属性：_proto__（原型链属性），属性值是当前实例所对应类的prototype

# 原型链

  • 它是JS中一种查找机制： 调用当前实例对象的某个属性（成员访问），先看自己私有属性中是否存在，存在调用的就是自己私有的；不存在， 则默认按照proto找所属类 prototype 上的公有属性和方法；如果还没有，再基于 prototype 上的proto继续向上级查找，直到找到 Object.prototype 为止 ---> 这种查找机制就叫原型链
  • 基于原型链的查找机制，需找的属性，是否为私有还是公有，都是相对的。
let ary1 = [10, 20]
ary1.push(100)
/**
 * 原理：ary1这个数组实例，基于原型链查找机制，
 * 找到了Array.prototype上的push方法，并且
 * 把push方法执行（方法执行的作用是把100放到ary1的末尾）
 */
console.log(ary1)

function Fn () {
  this.x = 100
  this.y = 200
  this.say = function () {
    console.log(this.x)
  }
}
Fn.prototype.say = function () {
  console.log(this.y)
}
Fn.prototype.eat = function () {
  console.log(this.x + this.y)
}
Fn.prototype.write = function () {
  this.z = 1000
}
let f1 = new Fn()
// 构造函数使用new关键字，带不带括号，函数都是执行的
f1.say()
//=> this:f1  => console.log(f1.x) => 100;
f1.eat()
//=> this:f1  => console.log(f1.x + f1.y) => 300;
f1.__proto__.say()
//=>this:f1.__proto__ => console.log(f1.__proto__.y) => undefined
Fn.prototype.eat()
//=> this: Fn.prototype => console.log(Fn.prototype.x + Fn.prototype) => NaN
f1.write()
//=> this:f1 => f1.z = 100 =>给私有f1设置一个私有的属性z = 1000
Fn.prototype.write()
//=> this: Fn.prototype => // Fn.prototype.z = 100;
//   =>给原型上设置一个属性z = 1000（属性是实例的公有属性）

# 基于内置类的原型扩展方法

  • 1. 内置类的原型上会默认存在很多常用的方法
    • Array.prototype / String.prototype / Object.prototype ... 这些方法实例都可以调用，但是内置的方法不一定能满足所有的开发需求，有很多方法需要自己来编写
  • 2. 向原型上扩展的方法，调用起来比较的方便
    • 使用语法：实例.方法()
    • 方法执行的时候，方法中的 this 就是当前处理的实例。
        1. 自己扩展的方法不能影响原有内置的方法（我们自己设置的方法最好加前缀：my）
        1. this 的结果一定是对象数据类型值，所以向基本数据类型的原型上扩展方法，方法被执行的时候，方法中的 this 不再是基本类型，但是还按照原始的方法处理即可（[[因为在运算的时候，会隐式调用对象的 valueOf 方法，返回它的原始值，而原始值就是之前的基本数据类型值]]）
        1. 如果返回的结果依然是当前类的实例，还可以继续调用当前类原型上其它的方法（如果不是自己类的实例，可以掉用其它类原型上的方法） => “链式写法”
    1. 对于一个对象来说，它的属性方法（私有的/公有的）存在“枚举”的特点： 在使用for...in循环的时候是否可以遍历到，能遍历到的是可枚举的，不能遍历到的是不可枚举的(内置类原型上自己扩展的方法是可枚举的)
    // obj.__proto__=Object.prototype;
// AAA是可枚举的属性了
Object.prototype.AAA = function AAA () {}
let obj = {
  name: 'Deivo',
  age: 18
}
for (let key in obj) {
  // 在遍历的时候，对于原型上扩展的公共属性方法，
  // 我们过滤掉，只遍历对象中私有的属性方法（必须
  // 是可枚举的）
  if (!obj.hasOwnProperty(key)) break
  console.log(key) // name, age
  // 如果不判断就会把 AAA 遍历到
}

# 重构类的原型

  • 重构类的原型：让某个类的原型指向新的堆内存地址（重定向）

    • 问题 1： 重定向后的空间中不一定有 constructor 属性（只有浏览器默认给 prototype 开辟的堆内存中才存在 constructor）,这样导致类和原型机制不完整；所以需要我们手动在给新的原型空间设计 constructor 属性：
    • 问题 2： 在重新执行之前，我们需要确保原有原型的堆内存中没有设置属性和方法，因为重定向后，原有的属性和方法没啥用了（如果需要克隆到新的原型堆内存中，我们还需要额外的处理）=>内置类的原型，由于担心这样的改变会让内置的方法消失，所以禁止了我们给内置类原型的空间重定向，例如 Array.prototype = {...}这样是没有用，如果想加方法 Array.prototype.xxx = function(){...}可以这样处理
function Fn() {
    // ...
}
// 1. 批量给原型设置属性方法的时候：重构类的原型
Fn.prototype = {
    constructor: Fn,
    getA: function () {},
    getB: function () {}
    ...
}

// 2. 批量给原型设置属性方法的时候：设置别名
let proto= Fn.prototype;
proto.getA = function() {}
proto.getB = function() {}
proto.getC = function() {}
proto.getD = function() {}

# 内置类 Function

  • Function 每一个函数都是 Function 的一个实例。
  • Function 原型是一个匿名函数 anonymous 但是相关操作和其他原型对象没有任何区别。
typeof Function.prototype // "function"
  • Function 的原型链(__proto__)指向 Function 的原型(prototype)
Function.prototype === Function.__proto__ // true
  • 每一个函数（包含内置类）都是内置类 Function 的一个实例，所以它们的proto一定指向是 Function.prototype.
  • 每一个对象（包含原型对象等）都是 Object 的实例，所以最后他们基于proto一定可以找到 Object.prototype
  • Object 作为一个类（一个函数）它是 Function 的一个实例；Function 虽然是函数（类）但它也是一个对象，所以他也是 Object 的一个实例；
Object.__proto__.__proto__ === Object.prototype // true;
  • 总结： 在 JS 中的任何实例（除了值类型的值）最后都是可以基于自己的proto找到Object.prototype，也就是所有的值都是 Object 的实例，从而得出 万物皆为对象。
